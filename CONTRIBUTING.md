# Contributing to SoundSpot

We love your input! We want to make contributing to this project as easy and transparent as possible, whether it's:

- Reporting a bug
- Discussing the current state of the code
- Submitting a fix
- Proposing new features

## We Speak English!
Even though our documentation and code comments are primarily in French (as we deploy mostly in francophone FabLabs), **we fully accept and encourage bug reports, feature requests, and Pull Requests in English.**

## How to report a bug
Please open an Issue on GitHub. If you can, provide:
* The version of the Raspberry Pi you are using.
* The output of the `check` command.
* A clear description of the expected behavior versus what actually happened.

## Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, ensure it works on a Raspberry Pi Zero 2W.
3. Keep your PRs focused on a single feature or bugfix.

## Code of Conduct
By participating in this project, you agree to maintain a respectful and harassment-free environment for everyone.
